3-vazifa: **Tizim administratori tomonidan yozib olingan tarmoq trafiklari orqali buzg‘unchining hatti-harakatlarini aniqlash (2000 ball)**

Bu vazifa sizga `.pcap` yoki `.pcapng` fayl ko‘rinishida berilgan tarmoq trafigini tahlil qilishni talab qiladi. Buzg‘unchi C2 (Command & Control) server bilan aloqa qilgan, ma’lumot o‘g‘irlagan, lateral movement qilgan yoki tarmoq ichida skanerlagan bo‘lishi mumkin. Sizning vazifangiz – trafik ichidan anomaliyalarni topish, IOCs (Indicators of Compromise) chiqarish va buzg‘unchining harakatlarini rekonstruksiya qilish.

---

## 1. ASOSIY TUSHUNCHA: PCAP NIMA?

PCAP (Packet Capture) – tarmoq orqali o‘tgan barcha paketlarning to‘liq yozuvi. Har bir paketda:
- Aniq vaqt (timestamp)
- Source IP, destination IP
- Portlar
- Protokol (TCP, UDP, ICMP, HTTP, DNS, TLS va b.)
- Payload (agar shifrlanmagan bo‘lsa, fayl yoki buyruq)

**Musobaqada sizga 3 turdagi PCAP berilishi mumkin:**
1. **To‘liq capture** – barcha paketlar (eng ko‘p ma’lumot)
2. **Filterlangan capture** – faqat ma’lum IP yoki port bo‘yicha
3. **Zeek/Suricata loglari** – metadata (conn.log, http.log, dns.log) + original PCAP

---

## 2. TAHLIL BOSQICHLARI (KALI LINUX DA)

### 2.1. Birinchi qarash – PCAP haqida umumiy ma’lumot

```bash
# Fayl hajmi, paket soni, vaqt oralig‘i
capinfos capture.pcap

# Protokol hierarxiyasi (qaysi protokol qancha)
tshark -r capture.pcap -q -z io,phs

# Eng ko‘p traffic yuborgan IP’lar (top talkers)
tshark -r capture.pcap -q -z endpoints,ip

# TCP conversationlar
tshark -r capture.pcap -q -z conv,tcp
```

**Nimaga qaraysiz:**
- Agar HTTPS (443) yoki DNS (53) juda ko‘p bo‘lsa – tunnelling bo‘lishi mumkin
- Agar kutilmagan portlar (4444, 8080, 1337) ko‘p bo‘lsa – C2 portlari
- Agar bitta IP ga juda ko‘p connection bo‘lsa – beaconing yoki exfiltration

---

### 2.2. DNS tahlili – DGA, tunnelling, C2 domain

Buzg‘unchilar ko‘pincha DNS orqali C2 manzilini topadi yoki ma’lumot o‘g‘irlaydi.

```bash
# Barcha DNS query’lar
tshark -r capture.pcap -T fields -e frame.time -e dns.qry.name -e dns.a -Y "dns.flags.response == 1"

# DNS tunnelling (katta TXT yoki uzun subdomain)
tshark -r capture.pcap -Y "dns.resp.type == 16 and dns.resp.len > 100"

# DGA aniqlash – uzun va entropiyasi yuqori domainlar
tshark -r capture.pcap -T fields -e dns.qry.name -Y "dns.flags.response == 0" | \
  awk -F'.' '{print $1}' | while read sub; do
    echo -n "$sub: "
    echo $sub | grep -o . | sort | uniq -c | awk '{s+=($1*log($1))}END{print -s}'
  done | awk '$2 > 3.5 {print}'
```

**Ko‘rsatkichlar:**
- Uzun, tasodifiy ko‘rinishdagi subdomainlar → DGA
- DNS TXT yoki MX recordlarida katta payload → DNS tunnelling
- Bir necha marta so‘ralgan, lekin hech qachon resolv qilinmagan domainlar → DGA

---

### 2.3. HTTP/HTTPS tahlili – C2, payload download, exfiltration

HTTP ko‘pincha C2 uchun ishlatiladi, chunki firewalllar odatda 80/443 ni ochiq qoldiradi.

```bash
# Barcha HTTP request’lar
tshark -r capture.pcap -T fields \
  -e frame.time -e ip.src -e ip.dst -e http.host \
  -e http.request.method -e http.request.uri -e http.user_agent \
  -Y "http.request"

# POST data (exfiltration yoki C2 command)
tshark -r capture.pcap -Y "http.request.method == POST" -V | grep -i "cmd\|exec\|whoami\|powershell"

# Fayl download (exe, ps1, dll)
tshark -r capture.pcap -Y "http.response and tcp.len > 10000" -T fields -e http.host -e http.request.uri

# User-Agent anomaliyalari (normal brauzer emas)
tshark -r capture.pcap -Y "http.request" -T fields -e http.user_agent | sort | uniq -c | sort -nr
```

**Ko‘rsatkichlar:**
- `POST /gate.php?id=123456` kabi URI – C2 beacon
- `User-Agent: Python-urllib`, `curl`, `Mozilla/5.0 (compatible; MSIE 10.0)` kabi g‘ayritabiiy agentlar
- Katta response body (payload download)
- `cmd=`, `exec=`, `powershell` kabi parametrlar

**TLS/HTTPS uchun (shifrlangan):**
```bash
# SNI (Server Name Indication) – qaysi domainga ulangan
tshark -r capture.pcap -T fields -e tls.handshake.extensions_server_name -Y "tls.handshake.type == 1"

# JA3 fingerprint (malware TLS signature)
tshark -r capture.pcap -T fields -e tls.handshake.ja3 -Y "tls.handshake.type == 1"
```

### 2.4. Beaconing aniqlash – C2 ning muntazam aloqasi

Malware odatda belgilangan intervalda C2 ga signal yuboradi (beacon).

**Scapy bilan Python skript:**
```python
from scapy.all import rdpcap, IP, TCP
from collections import defaultdict
import statistics

packets = rdpcap("capture.pcap")

connections = defaultdict(list)
for pkt in packets:
    if IP in pkt and TCP in pkt:
        if pkt[TCP].flags & 0x02:  # SYN flag
            dst = f"{pkt[IP].dst}:{pkt[TCP].dport}"
            connections[dst].append(float(pkt.time))

for dst, times in connections.items():
    if len(times) >= 5:
        intervals = [times[i+1] - times[i] for i in range(len(times)-1)]
        avg = statistics.mean(intervals)
        stdev = statistics.stdev(intervals) if len(intervals) > 1 else 0
        jitter = (stdev / avg * 100) if avg > 0 else 0
        
        if 10 < avg < 3600 and jitter < 30:
            print(f"[!] {dst}: {len(times)} connections, interval: {avg:.1f}s ± {stdev:.1f}s")
```

**Ko‘rsatkichlar:**
- Doimiy interval (masalan har 60 sekund)
- Jitter (variatsiya) 30% dan kam
- Paket hajmi kichik va doimiy

### 2.5. Suricata bilan IDS tekshiruvi

Suricata oldindan tayyor qoidalar bilan PCAP ni skanerlaydi.

```bash
# Suricata o‘rnatish (agar yo‘q bo‘lsa)

# PCAP ni tekshirish

# Chiqarilgan alertlarni ko‘rish
cat suricata_logs/fast.log
```

Suricata sizga avtomatik ravishda C2, malware, port scan, exploit kabi anomaliyalarni ko‘rsatadi.

### 2.6. Zeek (Bro) bilan metadata yaratish

Zeek PCAP dan struktur loglar chiqaradi: conn.log, http.log, dns.log, ssl.log, files.log.

```bash
sudo apt install zeek

# PCAP dan loglar yaratish
zeek -r capture.pcap

# Chiqarilgan loglarni tekshirish
cat conn.log | zeek-cut id.orig_h id.resp_h id.resp_p proto duration orig_bytes resp_bytes
cat http.log | zeek-cut host uri method user_agent
cat dns.log | zeek-cut query answers
```

Zeek loglari orqali:
- Qaysi IP qancha traffic yuborgan
- Qaysi domainlarga so‘rov bo‘lgan
- Fayl transferlari (files.log)
- TLS sertifikat ma’lumotlari

## 4. DETECT QILINADIGAN TTP’LAR

| TTP | Qanday topiladi |
|-----|---------------|
| **C2 Beaconing** | `tshark -z conv,tcp` + interval analysis; doimiy interval, kichik paket |

| **Credential Theft** | HTTP basic auth, FTP login, telnet password plaintext |

## 6. AMALIY MISOL (HAQIQIY MUSOBAQADAN)

Sizga `traffic.pcap` berilgan. Siz quyidagi savollarga javob topishingiz kerak bo‘ladi:

1. **Attacker IP qaysi?**
   ```bash
   tshark -r traffic.pcap -T fields -e ip.src | sort | uniq -c | sort -nr
   ```
   Eng ko‘p traffic yuborgan IP – attacker yoki compromised host.

2. **C2 server IP yoki domain?**
   ```bash
   tshark -r traffic.pcap -Y "http.request" -T fields -e http.host | sort | uniq -c
   ```

3. **Qanday fayl yuklangan?**
   ```bash
   tshark -r traffic.pcap --export-objects http,./http_files/
   sha256sum ./http_files/*
   ```

4. **Qanday credentiallar o‘g‘irlangan?**
   ```bash
   tshark -r traffic.pcap -Y "http.request.method == POST" -V | grep -i "user\|pass\|login"
   ```

5. **Beaconing interval qancha?**
   Yuqoridagi scapy script bilan hisoblang.

6. **Attack timeline qanday?**
   ```bash
   tshark -r traffic.pcap -T fields -e frame.time -e ip.src -e ip.dst -Y "http.request" | head -20
   ```

## 4. DISK IMAGE ICHIDAGI PCAP

Agar sizga disk image (E01, dd, raw, vmdk) berilgan bo‘lsa va PCAP fayl shu image ichida (masalan `C:\captures\traffic.pcap`) bo‘lsa:

### 4.1. Image ni mount qilish (Kali)

**E01 fayl:**
```bash
sudo ewfmount image.E01 /mnt/ewf
```
`/mnt/ewf/ewf1` – bu raw image.

**Raw/dd fayl:**
```bash
sudo fdisk -lu image.raw   # partitsiya offset ni topish
# Masalan: start sector = 2048, sektor hajmi 512 -> offset = 2048*512 = 1048576
sudo mount -o ro,loop,offset=1048576 image.raw /mnt/win
```

**VMDK:**
```bash
sudo modprobe nbd
sudo qemu-nbd -c /dev/nbd0 image.vmdk
sudo mount /dev/nbd0p1 /mnt/win
```

### 4.2. PCAP faylni qidirish
```bash
find /mnt/win -name "*.pcap" -o -name "*.pcapng" 2>/dev/null
```
Agar topilsa, masalan `/mnt/win/Users/Admin/Desktop/capture.pcap` bo‘lsa, uni `/tmp` ga ko‘chiring:
```bash
cp /mnt/win/Users/Admin/Desktop/capture.pcap /tmp/
```

### 4.3. Tahlilni davom ettirish
Endi fayl Kali da mavjud, yuqoridagi Kali usullarini (tshark, zeek, suricata, NetworkMiner) qo‘llaysiz.

### 4.4. Agar image ichida Windows bo‘lsa va siz Windows asboblari bilan tahlil qilmoqchi bo‘lsangiz
Image ni Windows virtual mashinaga mount qilishingiz mumkin (masalan VMware Workstation da .vmdk ni to‘g‘ridan-to‘g‘ri qo‘shish). Keyin Windows ichida Wireshark, NetworkMiner bilan tahlil.
